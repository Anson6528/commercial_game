# 星际贸易网页小游戏 — 界面设计改进指导文档

> **版本**: v1.0  
> **目标**: 明确当前界面的问题，定义期望的视觉风格与交互规范，供前端 Agent 在重构时严格遵循。

---

## 一、当前界面核心问题诊断

### 1.1 主界面（星图界面）

| 问题项 | 现状描述 | 期望改进 |
|---|---|---|
| **布局割裂** | 星图、货舱、信息栏被独立方框包围，彼此割裂，缺乏整体感 | 星图应**铺满整个视口**，其他 UI 元素以**悬浮层（Overlay）**形式浮于星图之上 |
| **星图被囚禁** | 星图被限制在固定矩形区域内，四周留有大量死空间 | 星图作为**背景画布**，无边框限制，可自由呼吸 |
| **图标过小** | 各功能图标尺寸偏小，在复杂背景下辨识度低 | 图标尺寸需**明显放大**，确保远距离可清晰辨识 |
| **按钮缺乏质感** | 所有按钮均为简单圆角矩形，无科技感和点击反馈 | 按钮需具备**工业控制面板质感**，有明确的按下/悬停状态 |
| **信息层级混乱** | 金钱、时间、通缉等级等信息挤在顶部一条细窄的横栏中 | 信息需按功能分组，以**模块化悬浮面板**呈现，留出呼吸空间 |

### 1.2 市场界面（站点市场）

| 问题项 | 现状描述 | 期望改进 |
|---|---|---|
| **列表感过重** | 商品以简单列表形式排列，像 Excel 表格 | 商品应以**卡片（Card）**形式呈现，每张卡片包含图标、名称、价格、库存，视觉丰富 |
| **价格显示平淡** | 价格仅为纯数字，无涨跌视觉提示 | 价格需配合**颜色编码**（绿色=低价/可买入，红色=高价/可卖出）和**趋势箭头** |
| **货舱对比弱** | 右侧货舱与左侧市场列表视觉权重相同，主次不分 | 市场列表为**主操作区**，货舱为**辅助信息区**，通过尺寸和透明度区分层级 |
| **交互反馈缺失** | 点击商品无选中态，购买按钮无按下感 | 需有明确的**选中高亮**、**按钮按下动画**、**数量变化动效** |

### 1.3 交易界面

| 问题项 | 现状描述 | 期望改进 |
|---|---|---|
| **简陋的表单感** | 交易界面像网页表单，缺乏游戏感 | 应设计为**沉浸式交易终端**，带有扫描线、数据刷新动效 |
| **数量调节枯燥** | +/- 按钮为简单符号，无手感 | 调节器应像**机械旋钮**或**数字拨盘**，带有阻尼感 |
| **支出显示不醒目** | "预计支出" 文字与数字混排，不直观 | 关键数字需**放大加粗**，配合**颜色警示**（支出为红色，收入为绿色） |
| **缺乏交易仪式感** | 点击购买后直接消失，无确认感 | 需有**交易确认动画**（如数据流闪过、货物图标飞入货舱） |

## 二、参考基准与风格定义

### 2.1 布局参考：游戏《群星》（Stellaris）

**参考图四（群星星图界面）的核心特征**：

- **全屏星图为底**：整个屏幕就是宇宙，UI 元素像飞船 HUD 一样悬浮在太空之上
- **边缘信息带**：顶部资源条、右侧通知/舰队列表、底部工具栏，均**贴边悬浮**，不遮挡中心视野
- **非对称设计**：信息面板大小不一，根据重要性自然分布，而非机械等分
- **半透明遮罩**：面板背景使用**深色半透明 + 轻微模糊**，既能看清背后星空，又能保证文字可读
- **角标装饰**：面板边缘有**斜切角**或**几何分割线**，避免纯矩形的呆板感

**参考图五（群星贸易界面）的核心特征**：

- **分栏对比布局**：左右两侧分别为己方和对方资源，中间为交易详情，逻辑清晰
- **资源条可视化**：资源不仅有数字，还有**横向条形图**显示数量占比
- **协议条款可视化**：交易条款像**合同条款**一样逐条列出，可勾选/取消
- **确认按钮突出**：底部确认按钮**宽大醒目**，带有发光边框

### 2.2 风格参考：鹰角网络（Hypergryph）UI 设计语言

基于《明日方舟》《终末地》等产品的 UI 设计分析，提取以下核心特征供本项目采用：

#### A. 色彩体系

| 用途 | 色值建议 | 说明 |
|---|---|---|
| **主背景** | `#0a0e1a` 或 `#0d1117` | 极深蓝黑，接近纯黑但带有一丝冷色调，营造太空深邃感 |
| **面板底色** | `rgba(13, 17, 28, 0.85)` | 半透明深色，配合 `backdrop-filter: blur(8px)` 实现毛玻璃效果 |
| **主强调色** | `#00d4ff`（ cyan 青） | 科技感主色，用于按钮、选中态、重要数字 |
| **辅强调色** | `#ff6b35`（橙红） | 用于警告、通缉等级、危险提示 |
| **成功/低价** | `#00e5a0`（青绿） | 买入推荐、价格低位、操作成功 |
| **危险/高价** | `#ff4757`（红） | 卖出推荐、价格高位、通缉警告 |
| **文字主色** | `#e0e6ed`（冷白） | 正文文字，避免纯白刺眼 |
| **文字次色** | `#8b95a5`（灰蓝） | 次要信息、标签、禁用态 |
| **边框/分割线** | `rgba(0, 212, 255, 0.3)` | 青色半透明边框，勾勒面板轮廓 |

#### B. 字体规范

- **标题字体**：使用**思源黑体 Heavy** 或类似的无衬线黑体，字间距适当收紧（`letter-spacing: -0.02em`），营造紧凑科技感
- **正文字体**：思源黑体 Regular / Medium，行高 1.5，确保长文本可读性
- **数字字体**：使用等宽字体（如 `Roboto Mono`、`JetBrains Mono` 或中文等宽），确保价格、数量对齐
- **装饰字体**：关键标题（如站点名、游戏标题）可加入**轻微发光效果**（`text-shadow: 0 0 10px rgba(0, 212, 255, 0.5)`）

#### C. 面板与边框设计

- **拒绝纯圆角**：采用**微圆角（2-4px）**或**斜切角（Chamfer）**，避免过度圆润的卡通感
- **边框样式**：使用 **1px 青色半透明实线** 或 **内侧发光边框**（`box-shadow: inset 0 0 0 1px rgba(0, 212, 255, 0.3)`）
- **角标装饰**：面板四角可添加**小型几何装饰**（如 L 形角标、三角形切角），增强工业感
- **内阴影/外发光**：重要面板使用**轻微外发光**（`box-shadow: 0 0 20px rgba(0, 212, 255, 0.1)`）提升层级

#### D. 图标设计规范

- **尺寸要求**：工具栏图标 ≥ 32×32px，货舱物品图标 ≥ 48×48px，星图节点图标 ≥ 64×64px
- **风格统一**：全部使用**线性图标（Line Icon）**或**面性图标（Fill Icon）**，保持一套风格
- **状态反馈**：图标需有**默认态**、**悬停态**（发光/放大 110%）、**按下态**（缩小 95% + 变暗）、**禁用态**（灰度 + 降低透明度）
- **动效**：悬停时图标**轻微上浮**（`translateY(-2px)`）并**散发微光**，点击时有**涟漪扩散**效果

#### E. 按钮设计规范

- **形状**：采用**矩形带微斜切角**或**六边形**，拒绝标准圆角矩形
- **层级区分**：
  - **主按钮**：青色填充 + 白色文字，带内阴影和底部光晕
  - **次按钮**：透明背景 + 青色边框，悬停时背景填充半透明青色
  - **危险按钮**：橙红边框/填充，用于卖出、放弃、通缉相关操作
- **点击反馈**：按下时按钮**下沉 2px**，内部出现**扫描线划过**动画，持续时间 200ms
- **尺寸**：主操作按钮高度 ≥ 48px，宽度根据内容自适应但 ≥ 120px，确保移动端可轻松点击

#### F. 背景与质感

- **噪点纹理**：在纯色背景上叠加**5% 透明度的噪点纹理**（noise texture），避免色块过于平坦单调
- **扫描线效果**：部分面板（如交易终端、货舱扫描）可加入**微弱扫描线**（scanline），增强 CRT 监视器感
- **网格底纹**：星图背景可叠加**极淡的六边形或方形网格**，营造战术地图感
- **粒子氛围**：星图背景可加入**缓慢飘浮的微尘/星光粒子**，提升画面呼吸感

#### 关键设计要点

1. **顶部信息带**
   - 背景：`rgba(10, 14, 26, 0.7)` + `backdrop-filter: blur(12px)`
   - 边框：底部 1px 青色半透明线
   - 各信息模块之间用**竖直分割线**隔开，分割线带微弱发光
   - 金钱数字使用**等宽字体**，字号 24px，带轻微青色发光
   - 通缉等级（WANTED）使用**橙红色**，带闪烁警告动画（间隔 2s）

2. **星图节点（站点图标）**
   - 尺寸：基础 64×64px，悬停时放大至 80×80px
   - 形状：根据站点类型使用不同轮廓
     - 贸易港：**六边形** + 内部货船图标
     - 矿业站：**方形带斜切角** + 内部矿石图标
     - 研究站：**圆形** + 内部烧瓶图标
     - 军事港：**盾牌轮廓** + 内部战舰图标
   - 连线：站点间连线使用**虚线**，颜色 `#00d4ff` 30% 透明度，悬停时变实线并高亮
   - 选中态：当前所在站点外围有**旋转的虚线圆环**（类似雷达扫描）

3. **右侧货舱面板**
   - 可收起/展开，收起时显示为右侧边缘的**竖条标签**（带货物数量徽标）
   - 货物卡片：横向排列，左侧图标（48×48），中间名称+数量，右侧单价
   - 违禁品（走私艺术品）图标带**红色警告角标** ⚠，边框为橙红色
   - 空仓位显示为**虚线边框的占位卡片**，提示货舱容量

4. **底部资源行情栏**
   - 每个资源为一个**微型卡片**，宽度均分
   - 图标上方，名称下方，百分比用**横向微型条形图**表示
   - 颜色编码：>70% 为青色，30-70% 为黄色，<30% 为红色
   - 点击可展开**详细行情弹窗**

5. **底部中央操作栏**
   - 按钮采用**六边形**或**矩形斜切角**设计
   - 每个按钮包含：**快捷键提示**（如 [M]）+ **图标** + **文字**
   - 按钮间距 16px，整体有**外发光**效果
   - 悬停时按钮**上浮 4px**，内部出现**扫描线动画**

#### 关键设计要点

1. **商品卡片设计**
   - 尺寸：宽度自适应（最小 200px），高度 160px
   - 背景：`rgba(13, 17, 28, 0.6)`，边框 1px 青色半透明
   - 结构：顶部图标区（64×64，居中）、中部名称（16px，白色）、底部价格（20px，等宽字体）
   - 价格颜色：低于均价显示**青绿色**（机会），高于均价显示**红色**（风险）
   - 库存条：卡片底部有**微型进度条**显示库存剩余比例
   - 受限商品（如精密零件）：卡片带**锁形图标**，边框为灰色，悬停显示"通缉等级限制"提示

2. **选中态**
   - 选中卡片边框变为**实线青色**，并添加**外发光**
   - 卡片**轻微放大**（scale 1.05），其他卡片**降低透明度**（0.6），形成聚焦效果
   - 选中后底部出现**操作栏**（买入/卖出按钮），从卡片底部滑出

3. **货舱对比区**
   - 背景透明度低于市场区（0.4 vs 0.6），形成主次层级
   - 货舱中已有商品，在市场列表中对应卡片会显示**"已持有"徽标**
   - 货舱中商品与市场价格对比：若当前市场价比持有均价低，显示**绿色向下箭头**（可补仓）；反之显示**红色向上箭头**

#### 关键设计要点

1. **价格显示**
   - 当前价格使用**超大字号**（48px），等宽字体，青色，带**呼吸灯动画**（透明度 0.8-1.0 循环）
   - 价格数字背景有**圆形扫描线**装饰，像雷达锁定目标

2. **相邻站点价格**
   - 以**小卡片**形式横向排列，像数据终端的监控窗口
   - 每个卡片显示：站点名 + 价格 + 涨跌幅
   - 涨跌幅用**箭头 + 百分比**，绿色为低于当前（可去那买），红色为高于当前（可去那卖）
   - 卡片边框颜色与涨跌对应：绿色边框 / 红色边框

3. **数量调节器**
   - 采用**机械拨盘风格**：中间数字显示区像显示屏，两侧 +/- 按钮像实体按键
   - 按钮形状：**方形带斜切角**，按下时有**下沉动画**和**内部扫描线**
   - 支持**直接输入**（点击数字区弹出数字键盘）和**拖拽滑动**（按住数字区左右滑动快速调节）
   - 数量变化时，数字有**滚动动画**（像老虎机数字滚动）

4. **支出预览**
   - 支出用**红色大字号**，收入用**绿色大字号**
   - 数字前有**+/- 符号**，符号与数字之间有**脉冲动画**
   - 背景有**微弱警告条纹**（斜向红白条纹，透明度 5%）

5. **确认按钮**
   - 按钮宽度占面板 80%，高度 56px，**六边形或矩形斜切角**
   - 默认态：青色填充，白色文字，带内阴影
   - 悬停态：亮度提升，**外发光增强**，图标（购物车）轻微跳动
   - 按下态：按钮下沉，颜色变深，内部出现**从上到下的扫描线**
   - 确认后：按钮变为**成功态**（绿色勾选 + "交易完成"），持续 1.5s 后关闭界面

## 四、全局交互与动效规范

### 4.1 悬浮面板行为

- **入场动画**：面板从对应边缘滑入（顶部面板从上滑入，右侧面板从右滑入），持续时间 300ms，缓动函数 `cubic-bezier(0.16, 1, 0.3, 1)`
- **收起动画**：向边缘滑出，持续时间 200ms，缓动函数 `ease-in`
- **拖拽行为**：面板标题栏可拖拽移动（仅限桌面端），拖拽时面板**透明度降低至 0.9**，释放后**弹性回归**原位或保持新位置

### 4.2 按钮点击反馈

- **按下**：`transform: scale(0.95) translateY(2px)`，颜色变暗，持续时间 100ms
- **释放**：`transform: scale(1) translateY(0)`，颜色恢复，若操作成功触发**涟漪扩散**（ripple）
- **涟漪**：从点击位置扩散的青色圆环，透明度 0.4 → 0，持续时间 600ms

### 4.3 数据更新动效

- **价格变动**：数字变化时使用**滚动动画**（像机械计数器），而非直接跳变
- **库存变动**：库存条宽度变化使用**弹性动画**（spring），而非线性插值
- **货舱更新**：新货物加入时，从交易界面**飞入**货舱面板，带有**抛物线轨迹**和**粒子拖尾**

### 4.4 警告与提示

- **通缉警告**：边框变为橙红色，每 2s 闪烁一次（透明度 1 → 0.5 → 1），伴随**微弱震动**（shake）
- **交易确认**：成功时屏幕边缘**闪过一道青光**，失败时闪过**红光**并伴随**错误提示音**

### 5.1 CSS 关键技术

```css
/* 毛玻璃面板基础类 */
.glass-panel {

  border: 1px solid rgba(0, 212, 255, 0.4);

.tech-button:hover {
  background: linear-gradient(180deg, rgba(0, 212, 255, 0.25) 0%, rgba(0, 212, 255, 0.1) 100%);
  box-shadow: 0 0 16px rgba(0, 212, 255, 0.3);
  transform: translateY(-2px);
}

.tech-button:active {
  transform: scale(0.95) translateY(2px);
}

/* 发光文字 */
.glow-text {
  text-shadow: 0 0 8px rgba(0, 212, 255, 0.6), 0 0 16px rgba(0, 212, 255, 0.3);
}

/* 扫描线动画 */
@keyframes scanline {
  0% { transform: translateY(-100%); }
  100% { transform: translateY(100%); }
}

.scanline-overlay::after {
  content: '';
  position: absolute;
  top: 0; left: 0; right: 0; height: 4px;
  background: linear-gradient(90deg, transparent, rgba(0, 212, 255, 0.4), transparent);
  animation: scanline 3s linear infinite;
  pointer-events: none;
}
```

### 5.2 性能注意事项

- `backdrop-filter` 在部分浏览器性能较差，建议对低端设备降级为**纯色半透明背景**
- 粒子效果使用 **Canvas 2D** 而非 DOM 节点，限制粒子数量 ≤ 100
- 动画优先使用 `transform` 和 `opacity`，避免触发重排（reflow）
- 使用 `will-change: transform` 对频繁动画的元素进行硬件加速提示

## 六、禁止事项清单

以下设计元素在当前项目中**严格禁止**使用：

- [ ] **标准圆角矩形按钮**（border-radius > 4px）
- [ ] **纯白背景或高亮背景**（所有面板必须为深色半透明）
- [ ] **系统默认字体**（必须使用指定科技风格字体栈）
- [ ] **无动效的界面切换**（所有页面切换必须有过渡动画）
- [ ] **静态无反馈的按钮**（所有可点击元素必须有 hover/active 态）
- [ ] **过小的图标**（任何功能图标不得小于 24×24px，推荐 ≥ 32×32px）
- [ ] **生硬的数值跳变**（所有数值变化必须有滚动/过渡动画）
- [ ] **等分布局强迫症**（拒绝所有区域等宽等高，必须按信息重要性非对称分布）

> **结语**：本项目的核心设计哲学是 **"星图为底，UI 为层，信息为光"**。所有界面元素都应像飞船 HUD 一样悬浮在宇宙之上，而非困在盒子里。科技感来源于**材质、动效、光影**的精细打磨，而非简单的"蓝色 = 科技"。请务必在实现时反复对比《群星》的沉浸感和鹰角网络的平面精致感，找到属于本项目的独特视觉语言。
