# 星际货运垄断者 — 前端开发详细设计文档

> **版本**: v1.0  
> **基于**: 国产数据库 openGauss 课程大作业  
> **游戏类型**: 非实时回合制太空贸易经营网页游戏  
> **目标平台**: 桌面浏览器（1920×1080 / 1440×900，无需移动端适配）  
> **核心机制**: 行动点制 + 世界回合推进 + 涟漪价格传播 + 垄断判定

---

## 1. 项目概述与核心设计转向

### 1.1 关键机制变更（相对于原 PDF 设计）
| 原设计 | 最终确定设计 | 原因 |
|--------|-------------|------|
| 10 分钟实时倒计时 | **行动点制**：每次行动消耗时间单位，时间耗尽结算 | 大幅降低前后端同步复杂度，非实时更易实现 |
| 定时批量事件（45秒/30秒） | **世界回合**：每次玩家行动后，系统自动推进世界回合，解锁/消失事件 | 与行动点制匹配 |
| 实时价格视图轮询 | **行动后增量更新**：玩家行动触发状态变化，前端接收增量补丁 | 避免无意义轮询，性能更优 |
| 全量状态同步 | **增量补丁同步**：后端只返回变化的部分 | 减少数据传输量，避免卡顿 |

### 1.2 游戏节奏
- **非实时**：只要玩家不操作，游戏世界完全静止。
- **行动消耗**：移动、交易、处理事件各消耗不同时间单位；查看信息/整理仓库不消耗。
- **胜负条件**：
  - **胜利**：单一商品在全星系持有率（货舱 + 全仓库）≥ 80%。
  - **失败**：资金 ≤ 0，或累计扣留时间超限，或行动点耗尽后未达成垄断。

## 2. 界面架构总览

游戏仅有 **三层界面**，无嵌套子页面（除模态弹窗外）：

```
登录界面 (Login Screen)
    ↓
大厅界面 (Lobby Screen) —— 开始游戏 / 查看分数 / 设置 / 退出
    ↓
游戏场景 (Game Scene) —— 核心玩法承载层
```

---

## 3. 游戏主场景布局（四分区）

游戏场景采用 **全屏星图背景 + 三层 HUD 叠加** 的"驾驶舱"布局：

```
┌─────────────────────────────────────────────┐
│ [上侧边栏] 资金 | 状态 | 宇宙年份 | 行动点/时间 │  [结束按钮] │
├────────────────────────┬────────────────────┤
│                        │ [右侧边栏]          │
│                        │ 货舱快捷栏          │
│                        │ (有库存商品+空槽)   │
│                        │                    │
│    [星图主场景]         │                    │
│  伪3D背景 / PixiJS     │                    │
│  WASD + 滚轮操控视角    │                    │
│  鼠标点击交互           │                    │
│                        │                    │
├────────────────────────┴────────────────────┤
│ [下侧边栏] 胜利进度(迷你进度条) | 当前站点名 │
└─────────────────────────────────────────────┘
```

### 3.1 上侧边栏（Top HUD Bar）
- **高度**: 固定 60-80px，占满屏幕宽度。
- **背景**: 半透明深色（rgba(10,15,30,0.85)）+ 底部 1px 科技蓝发光边框。
- **从左到右排列元素**：
  1. **资金 (Credits)**: `💰 12,450 CR`，资金变动时绿色（+）/红色（-）跳动动画。
  2. **状态 (Status)**: 当前玩家状态标签：`自由探索` / `航行中` / `交易确认` / `被扣留`。
  3. **宇宙年份 (Galactic Year)**: 静态装饰性数据，如 `Y-2149`，增加沉浸感。
  4. **行动点/时间 (Action Points)**: 如 `⏳ 剩余 47 单位`，用横向进度条或数字显示。
  5. **结束按钮**: 右上角，红色科技风格按钮 `结束本局` → 弹出确认 → 返回大厅。

### 3.2 右侧边栏 — 货舱快捷栏（Right Cargo Panel）
- **宽度**: 固定 200-240px，从顶部 HUD 下方延伸到底部 HUD 上方。
- **背景**: 半透明深色面板。
- **内容**: 
  - **标题**: `货舱 CARGO 14/80`
  - **格子布局**: 垂直排列的商品槽位，每个槽位高度约 60px。
  - **有库存槽位**: 显示商品图标 + 名称 + 数量 + 加权平均成本（如 `高能晶体 ×12 @ 340CR`）。
  - **空槽位**: 显示虚线边框 + `空仓` 占位符。
  - **违禁品标识**: 右上角 **黄色警告角标**（⚠）；当通缉等级升高时变为 **红色警告角标**（⛔）。
- **交互**: 点击任意槽位 → 从右侧滑出 **仓库管理宽面板**。

### 3.3 下侧边栏 — 信息栏（Bottom Info Bar）
- **高度**: 固定 100-120px，占满屏幕宽度。
- **背景**: 同上，半透明深色 + 顶部发光边框。
- **从左到右排列元素**：
  1. **胜利进度区（垄断面板）**: 
     - 8 种商品的迷你 UI 横向排列。
     - 每个迷你 UI：商品小图标 + 名称缩写 + 迷你进度条（0% → 80% → 100%）。
     - 进度条颜色：低于 50% 蓝色，50%-80% 橙色，≥80% 绿色闪烁（接近胜利）。
  2. **当前站点名**: 如 `📍 当前位置：阿尔法空间站 (安全等级: B)`。
  3. **鼠标悬停站点名**: 当鼠标悬停在星图某站点上时，此处显示 `悬停：贝塔贸易港 (预计移动消耗: 3单位)`。
- **左下角额外按钮**: `区域视图`（参考《文明6》势力范围），点击后星图视角切换为**区域垄断缩略图模式**。

### 3.4 星图主场景（Main Star Map）
- **渲染引擎**: PixiJS WebGL，全屏 Canvas 背景层。
- **视觉风格**: 
  - **背景**: 深色星空（深蓝/黑紫渐变）+ 远处星云粒子效果。
  - **空间站**: 伪 3D 建模节点，不同形状/颜色区分安全等级或功能（如球体=普通，环形=枢纽，菱形=高危）。
  - **航道**: 简洁线条连接相邻站点，正常为淡蓝色细线，被封锁时为红色虚线。
  - **当前位置高亮**: 玩家所在站点有呼吸灯效果（橙色光晕）。
- **操控方式**: 
  - `WASD`: 平移视角。
  - `鼠标滚轮`: 缩放视角（滚轮向上放大，向下缩小）。
  - `鼠标拖拽`: 也可平移视角（可选，与 WASD 并存）。

## 4. 核心交互流程详解

### 4.1 星图浏览与移动流程

#### 阶段 A：浏览与预览（玩家位于某站点时）
1. **相邻站点提示**: 
   - 从当前站点向所有**相邻可达站点**绘制**淡淡的橙色虚线箭头**。
   - 箭头旁悬浮显示：`消耗 3单位` + `航线状态: 正常`（或 `封锁`）。
2. **鼠标悬停**: 
   - 悬停在任意站点上 → 下侧边栏显示该站点名称 + 预计消耗。
   - 悬停站点本身高亮（外发光）。

#### 阶段 B：选择目标（第一次点击）
1. 点击相邻站点 A：
   - 当前站点、目标站点、连接航道、预计消耗时间全部打上**高亮边框/发光效果**。
   - 目标站点上方弹出确认 Tooltip：`再次点击确认移动 | 预计消耗: 3单位`。
2. 如果此时点击其他相邻站点 B：取消 A 的高亮，转为高亮 B。
3. 如果点击当前所在站点：打开**交易界面**（见 4.2）。

#### 阶段 C：确认移动（第二次点击同一站点）
1. 触发移动请求 → 前端发送 `POST /move` 到后端。
2. **界面锁定**: 所有 HUD 操作冻结（除视角操控外，但建议也冻结）。
3. **飞行动画**: 
   - 1-2 秒飞船沿航道飞行的跃迁/移动动画（粒子拖尾效果）。
   - 动画期间播放引擎音效（如有音频）。
4. **遭遇事件判定**: 
   - 动画播放期间，后端返回是否触发遭遇（30% 概率）。
   - **如果触发遭遇**: 在动画**播放中**跳出**全屏居中模态窗口**（遭遇事件界面）。
   - **动画不停止**，但遭遇模态强制覆盖在最上层，玩家必须处理完才能继续操作。
5. **状态结算顺序（严格）**: 
   - 先结算 **移动导致的状态变化**（到达新站点、扣除行动点、更新位置）。
   - 再结算 **遭遇事件导致的状态变化**（事件奖励/惩罚、额外行动点消耗）。
   - 两者**不可同时进行**，前端按顺序应用增量补丁。
6. **世界回合推进**: 移动完成后，系统自动推进一次世界回合（事件解锁/消失、概率更新、价格可能因全局事件变化）。
7. **界面解锁**: 玩家恢复自由操作。

### 4.2 交易流程

#### 阶段 A：打开交易界面
1. **触发**: 玩家位于某站点时，**点击当前所在站点**（不是相邻站点）。
2. **动画**: 
   - 交易界面从**屏幕底部快速上移**，最终成为**居中模态弹窗**。
   - 背景星图应用**高斯模糊滤镜**（blur 8-12px + 暗化遮罩）。
3. **加载态**: 
   - 界面上移至居中位置后，立刻显示**加载动画**（鹰角网络式科技加载：旋转几何图形 + "正在连接贸易网络..."）。
   - 原因：动态价格视图需要实时计算，后端查询需要一定时间。
4. **正式界面**: 加载完成后，分为左右两栏：
   - **左侧（60%）**: `本站商品市场` —— 该站点拥有的所有商品列表。
   - **右侧（40%）**: `飞船货舱` —— 复用货舱快捷栏的详细版，显示当前持有商品。

#### 阶段 B：商品列表（左侧）
- **商品 UI 卡片**（垂直列表，每个卡片高度约 80-100px）：
  - 商品图标 + 名称 + 类别标签。
  - **价格显示**: 大号字体显示当前动态价格。
  - **库存**: `库存: 45单位`。
  - **违禁品**: 右上角 **黄色警告角标** ⚠。
  - **价格变动**: 如果存在有意加价/减价（非正常市场波动），原价格**划删除线**，后面显示新价格（红色或绿色）。
  - **锁定状态**: 如果该商品因通缉等级被锁定，整个卡片**变灰（grayscale 80% + 透明度 50%）**，中央显示**锁图标** 🔒，不可点击。

#### 阶段 C：发起交易（点击商品卡片）
1. 点击非锁定的商品卡片 → 弹出**小交易详情界面**（覆盖在交易界面中央，或作为卡片展开动画）。
2. **小交易界面内容**：
   - **上部**: 价格趋势图（折线图，显示该商品最近 N 次交易/世界回合的价格变化）。
   - **中部**: 相邻站点价格对比（横向小条，显示相邻 3-4 个站点的当前价格，帮助玩家识别套利方向）。
   - **下部**: 
     - 数量输入框（数字输入，带 +/- 按钮）。
     - **买入 / 卖出 Switch 切换**（滑动开关或 Tab 切换）。
     - 预估总额：`预计支出: 3,400 CR`（实时随数量变化）。
3. **确认交易**: 
   - 点击 `执行交易` 按钮。
   - **无二次确认弹窗**，直接发送请求。
   - 按钮点击后立即进入 `loading` 状态，防止重复提交。

#### 阶段 D：交易结果与反馈
1. **成功**: 
   - 交易界面保持打开，但数据刷新（库存、资金、货舱更新）。
   - 星图背景中，相关站点节点触发**闪烁动画**（涟漪传播可视化，持续 1-2 秒）。
   - 上侧边栏资金数字跳动更新。
2. **失败**: 
   - 在交易界面顶部显示红色错误横幅：`交易失败：资金不足` / `货舱已满` / `库存不足` / `通缉状态限制`。
   - 不关闭界面，允许玩家调整数量重试。
3. **事务边界**: 交易涉及的所有数据库操作（库存、资金、日志、涟漪传播、通缉可疑度）处于同一事务，前端只需等待最终成功/失败结果。

### 4.3 仓库管理流程

#### 阶段 A：打开仓库面板
1. **触发**: 点击右侧边栏的货舱快捷栏任意槽位。
2. **动画**: 从**右侧滑出宽面板**（宽度约 500-600px），覆盖在星图和右侧边栏之上。
3. **背景**: 星图同样高斯模糊（或仅暗化）。

#### 阶段 B：面板内部布局
- **左栏（50%）**: `飞船货舱`
  - 显示当前货舱内所有商品的详细列表（图标/名称/数量/加权成本）。
  - 底部显示容量：`已使用 14/80`。
- **右栏（50%）**: `当前站点仓库`
  - 显示玩家存放在**这个特定站点**的所有商品。
  - 注意：每个站点的仓库是独立的，玩家必须到达该站点才能取出。
  - 每个商品显示：数量 + 已存放时间 + 预计取出税率。

#### 阶段 C：货物转移操作
1. **货舱 → 仓库**: 点击货舱商品 → `存入` 按钮 → 输入数量 → 确认。
   - **效果**: 立即生效，不消耗行动点，无税费。
2. **仓库 → 货舱**: 点击仓库商品 → `取出` 按钮 → 输入数量 → 确认。
   - **效果**: 
     - 自动计算并扣除税费（基于存放时间和当前税率）。
     - 如果通缉度高，可能触发**站点没收**（部分或全部货物被扣）。
     - 显示税费扣除提示：`-120 CR 仓储税`。
3. **交互方式**: 支持点击按钮转移，或拖拽（如实现成本允许）。
4. **关闭**: 点击面板外区域或 `X` 按钮 → 面板右滑收回。

### 4.4 遭遇事件流程

#### 触发时机
- 移动动画播放期间（1-2 秒时）判定，如果触发则跳出模态。
- 或玩家主动点击某些星图异常点时触发（如文档中的"航行遭遇"）。

#### 界面形态
- **全屏居中模态窗口**（参考《群星》事件窗口）。
- **背景**: 半透明黑色遮罩（rgba(0,0,0,0.7)），星图动画在遮罩后继续播放。
- **内容结构**:
  ```
  ┌─────────────────────────────┐
  │      [事件插画/动态背景]      │
  │      (科技风格抽象图形)        │
  ├─────────────────────────────┤
  │      事件标题：暗影走私商        │
  ├─────────────────────────────┤
  │  描述文本：一艘未标识的飞船向    │
  │  你发来通讯，提供一批高能晶体...  │
  ├─────────────────────────────┤
  │  [选项 A] 接受货物             │
  │    风险：+50 可疑度 | 收益：+20 单位违禁品 │
  ├─────────────────────────────┤
  │  [选项 B] 举报走私商           │
  │    风险：无 | 收益：+500 CR 奖金     │
  └─────────────────────────────┘
  ```
- **选项设计**: 
  - 每个选项下方明确标注**机会成本**（风险/收益预估）。
  - 部分选项可能因玩家状态不可用（如货舱已满时无法接受货物），显示为灰色并标注原因。
- **结果判定**: 
  - 玩家选择后，后端通过存储过程内的随机判定决定成功/失败。
  - 模态切换为结果页面：`成功！你获得了...` 或 `失败！你被...`。
  - 点击 `确认` → 关闭模态 → 应用状态变化 → 界面解锁。

## 5. 视觉与 UI 规范

### 5.1 视觉风格关键词
- **硬核科幻 HUD**: 数据感、功能性优先、扁平与轻 3D 结合。
- **鹰角网络式科技平面**: 直角/微圆角、细边框、高对比度文字、克制的发光效果。
- **色彩体系**:
  - 背景：深空蓝 `#0a0f1e` / 黑紫 `#0d0b14`
  - 主强调色：科技蓝 `#00d4ff` / 冷白 `#e0f7ff`
  - 危险/警告：橙 `#ff9f1c`（违禁品黄标）/ 红 `#ff2a6d`（通缉、错误、锁定）
  - 成功/利润：青绿 `#05ffa1`
  - 文字层级：标题纯白 `#ffffff` / 正文浅灰 `#a0b0c0` / 次要信息深灰 `#5a6a7a`
- **字体**: 
  - 英文/数字：等宽科技字体（如 `JetBrains Mono`, `Rajdhani`）。
  - 中文：无衬线黑体（如 `Noto Sans SC`, `思源黑体`）。
- **动效原则**: 
  - 界面切换：快速（200-300ms），使用 `cubic-bezier(0.4, 0, 0.2, 1)`。
  - 数据更新：数字跳动（计数动画），价格变动闪烁。
  - 涟漪传播：节点脉冲光环（从交易站点向外扩散 1-2 秒）。

### 5.2 各界面详细尺寸与样式

#### 交易界面（居中模态）
- **尺寸**: 宽度 900-1000px，高度 600-700px，屏幕居中。
- **圆角**: 4px（鹰角式微圆角）。
- **边框**: 1px 科技蓝半透明边框 + 外层微弱发光（box-shadow: 0 0 20px rgba(0,212,255,0.15)）。
- **背景**: 深色半透明面板（rgba(16,20,35,0.95)）。
- **加载动画**: 中央旋转几何体（两个嵌套正方形反向旋转）+ 脉冲文字。

#### 仓库面板（右侧滑出）
- **尺寸**: 宽度 550px，高度占满屏幕（减去上下 HUD 高度）。
- **动画**: 从 `translateX(100%)` 滑入到 `translateX(0)`，耗时 300ms。
- **背景**: 与交易界面一致。

#### 遭遇事件模态（全屏居中）
- **尺寸**: 宽度 700px，高度自适应（最小 400px）。
- **特殊装饰**: 顶部和底部有细线扫描动画（模拟数据加载）。

## 6. 游戏机制与数值框架（前端展示层）

### 6.1 行动点制（Action Points）
- **总行动点**: 每局固定总额（如 100 单位，具体数值待平衡调整）。
- **消耗基准**（暂定，后续可调）：
  - 基础移动：3-5 单位（随距离/航道等级变化）。
  - 交易操作：2 单位（无论买卖多少数量，一次交易动作固定消耗）。
  - 处理遭遇事件：1-3 单位（根据事件类型）。
  - 被扣留惩罚：直接扣除 5-10 单位（时间惩罚转化为行动点损失）。
- **免费动作**（不消耗行动点，不推进世界回合）：
  - 打开/关闭仓库面板。
  - 查看星图、浏览价格、悬停站点。
  - 在交易界面内切换商品、查看趋势图（但未执行交易）。

### 6.2 世界回合推进规则
- **触发条件**: 每次玩家完成一个**消耗行动点的动作**后，系统自动推进 1 个世界回合。
- **世界回合效果**:
  - 全局事件倒计时减 1（如航线封锁剩余回合 -1）。
  - 新事件可能解锁（市场冲击、航线危机）。
  - 某些事件概率表更新。
  - 通缉可疑度自然衰减（每 N 个世界回合 -5 分）。
- **前端表现**: 上侧边栏行动点数字减少；如果世界回合触发了全局事件，屏幕边缘闪过通知条（如 `⚠ 海盗封锁了阿尔法-贝塔航线`）。

### 6.3 税收与没收机制
- **仓储税**: 
  - 存入时：无费用。
  - 取出时：按 `数量 × 基础税率 × 存放回合数` 计算。
  - 税率受通缉等级影响：正常 5%，通缉等级 2 时 15%，等级 3 时 30% 或没收。
- **没收**: 
  - 高通缉状态下在某些安全等级高的站点取货，有概率被没收部分货物。
  - 前端显示：`⚠ 站点海关没收了你 5 单位高能晶体`。

### 6.4 垄断与胜利判定
- **持有率计算**: `(货舱持有 + 全星系仓库持有) / (市场流通总量 + 所有玩家持有)`。
- **前端展示**: 
  - 下侧边栏 8 个迷你进度条实时显示（每次行动后更新）。
  - 达到 80% 时该商品进度条绿色闪烁，并弹出 `🎉 垄断达成！` 横幅。
- **区域垄断**: 
  - 相邻 5 个站点中某商品持有率 ≥ 80%。
  - 点击左下角 `区域视图` → 星图切换为势力范围视图（被垄断区域染色，类似文明6）。

## 7. 状态同步与数据策略

### 7.1 增量补丁协议（Incremental Patch）
由于采用非实时行动点制，每次玩家行动后，后端只返回**变化的数据片段**，前端局部更新。

**请求格式**:
```json
POST /api/action
{
  "action_type": "trade", // move | trade | event_choice | warehouse_transfer
  "payload": { ... }
}
```

**响应格式（成功）**:
```json
{
  "status": "success",
  "action_cost": 2,
  "world_turn_advanced": true,
  "patch": {
    "player": {
      "credits": 12450, // 只传变化后的最终值
      "cargo": {
        "goods_id_3": { "quantity": 12, "avg_cost": 340 }
      },
      "action_points_remaining": 45
    },
    "station_inventory": {
      "station_5": {
        "goods_id_3": { "stock": 33, "base_price": 380 }
      }
    },
    "ripple_affected_stations": ["station_6", "station_7"], // 涟漪传播影响站点ID
    "wanted": {
      "suspicious_score": 85,
      "level": 1
    },
    "monopoly_progress": {
      "goods_id_3": 0.72
    },
    "global_events": [
      { "event_id": 7, "status": "expired" },
      { "event_id": 12, "status": "triggered", "affected_routes": [3, 4] }
    ],
    "encounter": { // 如果触发遭遇
      "encounter_id": 5,
      "title": "暗影走私商",
      "description": "...",
      "choices": [...]
    }
  }
}
```

**前端更新策略**:
- 维护一个全局 `gameState` 对象。
- 收到 patch 后，使用深合并（deep merge）将 patch 应用到 `gameState`。
- 触发对应的 UI 更新（如 `player.credits` 变化 → 上侧边栏数字跳动）。
- `ripple_affected_stations` 用于触发星图节点脉冲动画。

### 7.2 加载与错误状态
- **行动等待期**: 玩家发起行动后，前端进入 `isProcessing` 状态，屏蔽所有交互，显示等待指示器（如按钮 loading、屏幕中央微光晕）。
- **网络错误**: 如果请求超时或失败，显示 `连接中断，本局已放弃` → 强制返回大厅。
- **断线/刷新**: 
  - 课程作业场景下，**刷新浏览器或断网直接判定放弃**，清除本地状态，返回大厅。
  - 无需本地缓存恢复机制，简化实现。

## 9. 技术实现要点

### 9.1 前端技术栈建议
- **渲染引擎**: PixiJS v7+（星图、航道、飞船动画、节点脉冲效果）。
- **UI 框架**: React / Vue 3（HUD 层、模态窗口、数据面板）。
- **状态管理**: Zustand / Pinia（全局 gameState，支持增量 patch 的深合并）。
- **动画库**: GSAP（界面滑入滑出、数字跳动、加载动画）。
- **样式方案**: Tailwind CSS + 自定义 CSS 变量（科技蓝主色体系）。

### 9.2 PixiJS 与 DOM UI 的层级管理
```
Layer 0: PixiJS Canvas (全屏，z-index: 0)
  - 星空背景
  - 航道线条
  - 空间站节点
  - 飞船动画
  - 涟漪脉冲特效
Layer 1: DOM HUD Overlay (z-index: 10)
  - 上侧边栏
  - 右侧边栏
  - 下侧边栏
Layer 2: DOM Modals (z-index: 20)
  - 交易界面
  - 遭遇事件
  - 结算界面
  - 开发者面板
Layer 3: Loading & Toasts (z-index: 30)
  - 全局加载遮罩
  - 错误提示
  - 世界事件通知条
```

### 9.3 关键性能注意
- **星图节点**: 20 个站点 + 航道，数据量极小，PixiJS 无压力。
- **模糊滤镜**: 交易界面打开时，对 PixiJS Canvas 应用 CSS `filter: blur(8px)` 即可，无需 PixiJS 内部滤镜（性能更好）。
- **增量更新**: 避免每次行动后重绘整个星图，仅更新受影响的节点颜色/数据标签。
- **图片资源**: 8 种商品图标、空间站 3D 简模（可用 2D 精灵图 + 法线贴图伪 3D）、事件插画背景。

## 11. 接口契约草案（前后端对接参考）

### 11.1 核心端点
| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/game/start` | POST | 开局初始化，返回初始完整状态 |
| `/api/action` | POST | 通用行动接口（移动/交易/事件选择/仓库转移） |
| `/api/game/state` | GET | 仅在异常恢复时调用，课程作业中可忽略 |
| `/api/game/end` | POST | 主动结束本局，返回结算数据 |

### 11.2 行动类型枚举
```typescript
type ActionType = 
  | 'move'           // 移动飞船
  | 'trade'          // 买入/卖出
  | 'event_choice'   // 遭遇事件选择
  | 'warehouse_transfer'; // 货舱与仓库间转移
```

### 11.3 关键数据结构（前端 TypeScript 定义）
```typescript
interface GameState {
  player: {
    player_id: number;
    credits: number;
    current_station_id: number;
    cargo_capacity: number;
    cargo_used: number;
    action_points_remaining: number;
    wanted_level: number;
    suspicious_score: number;
    game_status: 'PLAYING' | 'WON' | 'LOST' | 'TIMEUP';
    cargo: Record<number, { quantity: number; avg_cost: number }>;
  };
  stations: Record<number, {
    station_id: number;
    name: string;
    x: number; y: number;
    security_level: number;
    independence_factor: number;
    goods: Record<number, { stock: number; base_price: number; current_price: number; is_locked: boolean }>;
  }>;
  routes: Array<{ from: number; to: number; distance: number; is_active: boolean; cost: number }>;
  warehouse: Record<string, { station_id: number; goods_id: number; quantity: number; stored_turns: number }>;
  monopoly_progress: Record<number, number>; // goods_id -> 持有率 0-1
  active_events: Array<{
    event_id: number;
    title: string;
    description: string;
    choices: Array<{ choice_id: number; text: string; consequence_hint: string }>;
  }>;
  world_turn: number;
}
```
