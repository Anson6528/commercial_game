# 星际贸易模拟游戏 - 后端框架说明

## 项目概述
后端负责：
- 处理前端请求（交易、移动、事件选择）
- 数据库事务协调与错误处理
- 提供经济计算、涟漪传播、通缉等级、垄断判定的 API
- 可选 WebSocket 推送实时状态

核心业务逻辑全部封装在 **OpenGauss 数据库** 中，后端仅作为 API 代理和事务边界控制。

## 技术栈
- **语言**：Python 3.10+
- **框架**：FastAPI 或 Flask
- **数据库**：OpenGauss
- **ORM**：SQLAlchemy 或 Psycopg2（可选）
- **异步支持**：Asyncio + WebSockets（可选）
- **任务调度**：APScheduler（用于定时垄断判定）

## 项目结构
    /backend
    │
    ├─ /app
    │ ├─ /api # HTTP路由
    │ ├─ /services # 服务层（封装交易、移动、事件逻辑调用）
    │ ├─ /db # 数据库连接、事务管理
    │ ├─ /models # ORM/数据库表映射
    │ ├─ /schemas # Pydantic数据验证
    │ ├─ /utils # 工具函数（时间、日志等）
    │ └─ main.py # 入口文件
    ├─ requirements.txt
    └─ alembic/ # 数据库迁移（可选）


## 开发要求
1. **可扩展性**：
   - API路由与服务逻辑分离
   - 数据库操作通过事务和存储过程封装
   - 支持多模块扩展，如新事件、新商品类型
2. **代码逻辑正确性**：
   - 交易、涟漪传播、通缉等级必须在同一事务内原子执行
   - 错误处理和异常回滚机制完善
3. **简洁性与可读性**：
   - 遵循 PEP8、类型注解清晰
   - 逻辑分层明确：路由 -> 服务 -> 数据库
4. **性能优化**：
   - 重量级计算放在数据库中执行（CTE、视图、触发器）
   - API只负责参数校验、事务边界、结果返回

## 开发任务示例
- 初始化 FastAPI 项目
- 配置数据库连接与事务管理
- 封装交易服务：提交交易、更新库存、触发涟漪
- 封装通缉服务：窗口函数计算可疑度
- 封装垄断判定服务：定时聚合 PlayerCargo + WarehouseInventory
- 编写 API 路由和数据验证
- 配置 WebSocket 推送实时价格和事件状态
- Mock 数据库操作进行接口测试
